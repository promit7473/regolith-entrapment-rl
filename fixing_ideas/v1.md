## Suggestion 1: Torque Anomaly Detection (Currently 0%)

### Problem
95% threshold is too strict — motors may never reach that even when buried in sand.

### Current Implementation
```python
# From entrapment_env.py
_TORQUE_ANOMALY_THRESH = 0.95  # 95% of max torque
_TORQUE_ANOMALY_STEPS = 10    # consecutive steps
```

### Ideas to Fix

| Approach | Description | Priority |
|----------|-------------|----------|
| Lower threshold | Try 80% or 85% of max torque | High |
| Relative threshold | Compare to running average instead of absolute max | Medium |
| Sustained duration | Already doing 10 steps, could increase to 15-20 | Low |
| Per-wheel anomaly | Flag if ANY wheel exceeds threshold (currently using mean?) | Medium |
| Check actual values | Log `drive_torque.max()` during entrapment to see actual range | High |

### Quick Diagnostic
Add logging to see actual torque values during entrapment:
```python
# In _get_rewards(), log max torque when entrap_flag is True
```

### Recommended Action
1. First: Log actual torque values to determine realistic range
2. Then: Adjust threshold to ~80% of observed max

### Ideas to Fix

| Approach | Description | Expected Effect |
|----------|-------------|-----------------|
| Reward normalization | Use running mean/std normalization via skrl | Stabilize learning |
| Increase `rew_rocking` | Currently 1.0 → try 2.0-3.0 | Stronger signal for escape behavior |
| Shaped escape reward | Progressive bonus at 0.25m increments instead of binary at 1.5m | Smoother reward landscape |
| Penalty for backward motion | Add `-v_x` penalty when `v_x < 0` | Discourage backing up |
| Curriculum-aware rewards | Increase escape bonus as sinkage increases | Scale difficulty with reward |

### Shaped Escape Reward Example
```python
# Instead of binary bonus at 1.5m
r_escape = 5.0 if dist > ESCAPE_DISTANCE else 0.0

# Use progressive shaping
r_escape = 0.0
for threshold in [0.25, 0.5, 0.75, 1.0, 1.25, 1.5]:
    if dist > threshold:
        r_escape += 1.0  # 1.0 bonus per 0.25m achieved
```

### Recommended Action
1. Increase `rew_rocking` to 2.0
2. Implement shaped escape reward
3. Run 200K steps to see if oscillation decreases naturally

## Suggestion 3: Escape Rate Drops to 0%

### Problem
Escape rate periodically drops to 0% during training.

### Analysis
This is actually **correct behavior** — curriculum increases sinkage, making escape harder. The agent needs to learn the harder task.

### Current Curriculum
```python
# From EntrapmentEnvCfg
dr_sinkage_range = (0.04, 0.10)  # 4cm to 10cm
# Curriculum linearly interpolates over training
```

### Ideas to Improve

| Approach | Description | Pros | Cons |
|----------|-------------|------|------|
| Smoother curriculum | Linear interpolation instead of discrete jumps | Smoother learning | Already linear |
| Curriculum as observation | Add normalized sinkage depth to obs | Agent knows difficulty | Privileged info |
| Success-based curriculum | Only increase sinkage when escape rate > 50% | Better pacing | More complex |
| Minimum sinkage floor | Don't go below 4cm (already doing this) | Prevents too-easy | — |

### Success-Based Curriculum Example
```python
# Only advance curriculum if agent is succeeding
if escape_rate > 0.5:
    self._curriculum_progress += 0.01
else:
    self._curriculum_progress -= 0.005  # Slight regression if failing
```

### Recommended Action
1. Current behavior is acceptable
2. Consider adding sinkage depth to observations for curriculum awareness
3. Monitor if escape rate recovers after drops (it should as agent learns)

### Ideas

| Parameter | Current | Suggested | Reason |
|-----------|---------|-----------|--------|
| `entropy_loss_scale` | 0.02 | 0.03-0.05 | Prevent premature convergence |
| `ROLLOUTS` | 32 | 48-64 | Better credit assignment |
| `learning_rate` | 3e-4 | 1e-4 to 5e-4 | Try lower for stability |
| `GRU_LAYERS` | 1 | 2 | More temporal depth |

### Asymmetric Actor-Critic
Give the critic privileged information that the actor doesn't have:
- Actual sinkage depth (ground truth)
- Ground truth entrapment state
- True terrain gradient

This helps the critic provide better value estimates without cheating the policy.

### Recommended Action
1. Increase `entropy_loss_scale` to 0.03
2. Increase `ROLLOUTS` to 48
3. Consider asymmetric critic after 200K baseline run
