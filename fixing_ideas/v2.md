## The Critical Reward Bug

### Problem: Slip Penalty Too High

**Per-step reward math** (dt = 0.02, 500 steps per episode):

| Term | Formula | Per Step | Per Episode | Notes |
|------|---------|----------|-------------|-------|
| r_progress | 3.0 Г— v_x Г— dt | +0.012 | +6.0 | v_x в‰€ 0.2 m/s in sand |
| p_slip (OLD) | 1.0 Г— slip Г— dt | -0.016 | -8.0 | slip в‰€ 0.8 on sand |
| Net forward driving | | **-0.004** | **-2.0** | вќЊ NEGATIVE! |

**Result**: Agent learns that driving forward = negative reward. Doing nothing = zero reward. So it learns to stop moving.

### Fix: Reduce Slip Penalty

```python
# Before
pen_slip = 1.0  # Too high вЂ” slip is unavoidable on sand

# After  
pen_slip = 0.3  # Reduced вЂ” slip is expected in granular terrain
```

**New per-step math**:

| Term | Formula | Per Step | Per Episode | Notes |
|------|---------|----------|-------------|-------|
| r_progress | 3.0 Г— v_x Г— dt | +0.012 | +6.0 | v_x в‰€ 0.2 m/s |
| p_slip (NEW) | 0.3 Г— slip Г— dt | -0.005 | -2.4 | slip в‰€ 0.8 |
| Net forward driving | | **+0.007** | **+3.6** | вњ… POSITIVE! |

**Result**: Agent now receives net positive reward for driving forward. The reward signal aligns with the task goal.

## Other Fixes Made

### 1. Torque Anomaly Detection Bug

**Problem**: 
- `clamp(0.0, 2.0)` killed negative torques в†’ half the wheels always read as zero
- `mean()` across 6 wheels diluted signal в†’ single stuck wheel wouldn't trigger

**Fix**:
```python
# Before
torque_ratio = (drive_torque / drive_effort_lim).clamp(0.0, 2.0)
mean_torque_ratio = torch.mean(torque_ratio, dim=-1)
is_anomalous = (mean_torque_ratio > self._TORQUE_ANOMALY_THRESH)

# After
torque_ratio = (drive_torque.abs() / drive_effort_lim).clamp(0.0, 2.0)
max_torque_ratio = torch.max(torque_ratio, dim=-1).values
is_anomalous = (max_torque_ratio > self._TORQUE_ANOMALY_THRESH)
```

**Result**: Torque anomaly now detects when ANY wheel is working hard (forward or reverse).

### 2. Sinkage Range (Kept Deep)

**Decision**: Keep sinkage at 8-15cm (80-150% burial)

**Reasoning**:
- This is a regolith entrapment research project
- Shallow sinkage defeats the purpose вЂ” nothing to escape from
- User observed escapes at deep sinkage в†’ task IS solvable
- Agent needs time to learn, not easier task
